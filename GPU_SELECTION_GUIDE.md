## 1. 模型训练需求分析

### 1.1 计算精度：纯 FP32

**关键发现：本模型全程使用 float32 精度，未使用混合精度（AMP）。**

代码证据：
- `train.py:48` — `background = torch.tensor(bg_color, dtype=torch.float32, device="cuda")`
- `scene/gaussian_model.py:32` — `trans = torch.zeros(..., dtype=torch.float, device="cuda")`
- 全代码库中无 `torch.cuda.amp`、`autocast`、`GradScaler` 的使用

**意义：** 这意味着：
- **单精度（FP32）TFLOPS 是决定训练速度的核心指标**
- 半精度（FP16）Tensor TFLOPS 在当前代码下无法被利用
- 未来若引入混合精度训练，Tensor TFLOPS 才有参考价值

### 1.2 显存需求

#### 每个高斯基元的内存占用

| 参数 | 维度 | 字节数（float32） |
|---|---|---|
| 位置（xyz） | 3 | 12 |
| SH DC 分量 | 1×3 | 12 |
| SH 高阶分量（degree=3） | 15×3 | 180 |
| 2D 缩放 | 2 | 8 |
| 四元数旋转 | 4 | 16 |
| 不透明度 | 1 | 4 |
| **小计（参数）** | **58** | **232 字节** |

训练时 Adam 优化器额外需要：
- 一阶动量（momentum）：≈ 232 字节/基元
- 二阶动量（variance）：≈ 232 字节/基元
- 梯度缓冲：≈ 232 字节/基元

**训练总计 ≈ 928 字节/基元（参数 × 4）**

#### 典型场景显存估算

| 高斯基元数量 | 参数 | 训练总计（含优化器） | 典型场景 |
|---|---|---|---|
| 100 万 | 0.22 GB | 0.88 GB | 小型物体 |
| 500 万 | 1.08 GB | 4.32 GB | 中等场景 |
| 1000 万 | 2.16 GB | 8.64 GB | 标准场景 |
| 5000 万 | 10.80 GB | 43.20 GB | 大型场景 |
| 1 亿 | 21.60 GB | 86.40 GB | 超大场景 |

**注意：** 以上仅为高斯参数与优化器状态的显存。实际训练还需要：
- 渲染中间结果（allmap 7通道、RGB图像）
- 反向传播计算图
- CUDA 光栅化核函数的临时缓冲区
- 多视图训练时 M 个视图的并行缓冲

#### README 中的官方要求

```

python run_360.py
# → 最低需要 40GB 显存

# 降低多视图数以减少显存
python train.py -s {data_path} --eval --white_background -m {save_path} --mv 6
# → 约 20-24GB 可运行
```

### 1.3 CUDA 依赖（硬性约束）

本模型依赖以下自定义 CUDA 核函数：

| 子模块 | 用途 | 依赖 |
|---|---|---|
| `diff-surfel-rasterization` | 2D-GS surfel 光栅化（级联 0+1） | **NVIDIA CUDA** |
| `diff-gaussian-rasterization` | 3D-GS EWA 溅射（级联 2） | **NVIDIA CUDA** |
| `simple-knn` | KNN 距离计算（初始化） | **NVIDIA CUDA** |

**硬性结论：必须使用 NVIDIA GPU。** 非 NVIDIA 架构（华为昇腾 CANN、摩尔线程 MUSA）
无法编译和运行这些 CUDA 核函数。

### 1.4 训练迭代量

| 阶段 | 迭代数 | 分辨率 | MV |
|---|---|---|---|
| 阶段 1（粗糙） | 3,000 | 1/8× | 48 |
| 阶段 2 | 3,000 | 1/4× | 24 |
| 阶段 3 | 3,000 | 1/2× | 12 |
| 阶段 4（精细） | 30,000 | 全分辨率 | 8 |
| **合计** | **~39,000** | — | — |

### 1.5 计算瓶颈分析

训练瓶颈按重要性排序：

1. **自定义 CUDA 光栅化核函数** — 前向+后向，逐高斯点、逐像素计算
2. **FP32 矩阵运算** — 协方差矩阵构建、SH 求值
3. **显存带宽** — 大量高斯参数的读写
4. **Adam 优化器更新** — 逐参数更新

由于训练全程 FP32，**CUDA 核心数和 FP32 TFLOPS 是最关键的性能指标**。

## 5. 训练时间估算模型

### 5.1 估算方法

由于本模型**全程 FP32**，训练时间主要与 FP32 算力成反比。
但需注意，CUDA 光栅化核函数的性能还受以下因素影响：

- **SM（流式多处理器）数量**：并行处理高斯基元和像素的能力
- **显存带宽**：大量高斯参数的读写速度
- **L2 缓存大小**：影响核函数的数据复用效率

为简化，我们以 **FP32 TFLOPS** 作为主要估算依据，并引入显存带宽修正因子。

### 5.2 基准参考

以 RTX 4090（82.58 TFLOPS）为基准，典型 Mip-NeRF360 场景的完整
多阶段训练（~39,000 迭代，MV=1 模式）约需 **1.5 小时**。

> 注：此为 `--mv 1` 下的估算（24GB 显存限制）。更高 MV 值的显卡训练时间可能更长
>（更多前向/后向传播），但每迭代获得更多信息，总迭代数可适当减少。

### 5.3 各显卡估算训练时间

以 RTX 4090 为基准（T_base = 1.5 小时），按 FP32 TFLOPS 比例估算：

```
T_est = T_base × (82.58 / FP32_TFLOPS_gpu)
```

| 显卡 | FP32 TFLOPS | 估算训练时间 | MV 模式 | 说明 |
|---|---|---|---|---|
| PRO 6000 | 126.0 | **~1.0 小时** | 完整 MV | FP32 最强 |
| RTX 5090 | 104.8 | **~1.2 小时** | MV≤6 | 显存限制需降 MV |
| RTX 4090 | 82.58 | **~1.5 小时** | MV≤4 | 基准 |
| L20 | 59.35 | **~2.1 小时** | 完整 MV | 48GB 可跑高 MV |
| H800 | 51.2 | **~2.4 小时** | 完整 MV | 80GB 无压力 |
| H20 | ~44.0 | **~2.8 小时** | 完整 MV | 96GB 最充裕 |
| RTX 3090 | 35.58 | **~3.5 小时** | MV≤4 | |
| A800-80GB | 19.5 | **~6.4 小时** | 完整 MV | FP32 较弱 |
| V100 | 15.7 | **~7.9 小时** | MV≤6 | 老架构，FP32 最弱 |

> **重要修正：** 高 MV 模式的显卡（如 H800、PRO 6000）每迭代处理更多视图，
> 单迭代时间更长，但多视图正则化更强，**可能**以更少迭代达到同等质量。
> 上述估算假设相同总迭代数，实际中高 MV 可能节省 20-30% 迭代。

## 附录 A：潜在优化方向

### A.1 引入混合精度训练

若在 `train.py` 中引入 `torch.cuda.amp` 混合精度训练，Tensor TFLOPS 将变得有意义：

| 显卡 | 当前 FP32 排名 | 混合精度排名（预计） | 变化 |
|---|---|---|---|
| H800 | 8/9 | **1/9** | 756 Tensor TFLOPS 将彻底释放 |
| PRO 6000 | 5/9 | **2/9** | 503.8 Tensor TFLOPS |
| A800-80GB | 9/9 | **3/9** | 312 Tensor TFLOPS 跳升 |
| RTX 5090 | 2/9 | **4/9** | |
| RTX 4090 | 1/9 | **5/9** | 性价比仍然出色 |

> **建议：** 如果训练量较大（数十个场景），可考虑先投入时间为代码添加 AMP 支持，
> 然后选择 H800/A800 类 Tensor Core 优势显卡，长期总成本可能更低。

### A.2 CUDA 版本兼容性

当前 `environment.yml` 指定 CUDA 11.6 + PyTorch 1.12.1。
较新的显卡（RTX 5090、H20、PRO 6000）可能需要更高版本的 CUDA：

| 显卡 | 最低 CUDA 版本 | 建议 |
|---|---|---|
| RTX 5090（Blackwell） | CUDA 12.8+ | 需升级 PyTorch 至 2.x + CUDA 12.8 |
| H20（Hopper） | CUDA 11.8+ | 需升级至 CUDA 11.8 或 12.x |
| PRO 6000（Blackwell/Ada） | CUDA 12.4+ | 需升级 |
| RTX 4090（Ada） | CUDA 11.8+ | 需升级至 CUDA 11.8 |
| RTX 3090（Ampere） | CUDA 11.1+ | ✅ 兼容 |
| V100（Volta） | CUDA 10.0+ | ✅ 兼容 |

> **注意：** 选择较新显卡时需要同步升级 CUDA 和 PyTorch 版本，
> 并重新编译自定义 CUDA 子模块。
